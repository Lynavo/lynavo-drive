# Stable Device New Folder Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the desktop sidecar create a new `received/` subfolder when the same physical phone logs in with a different account and reports a different device name, then package a `global-review` build number 58 release for iOS TestFlight and macOS DMG.

**Architecture:** Keep mobile logout/account switching non-destructive. Treat desktop `received/` folders as durable history buckets: same physical phone plus same preferred folder name may reuse a folder, but same physical phone plus a different preferred folder name creates or reuses that specific new-name folder. `手機同步空間` continues using the global received library so content across all received subfolders remains visible.

**Tech Stack:** Go sidecar with SQLite store, Jest/TypeScript React Native mobile tests, Xcode iOS TestFlight archive/upload, Electron Builder signed macOS DMG, root `pnpm release` profile flow.

```go
preferredDirName := preferredReceiveDirName(device)
```

Then update the candidate loop so it only considers same-stable rows whose `receive_dir_name` equals `preferredDirName`:

```go
dirName := *candidate.ReceiveDirName
if dirName != preferredDirName {
	continue
}
if !dirExists(receiveDir, dirName) {
	continue
}
if receiveDirReservedByDifferentStable(devices, dirName, device.ClientID, stableID) {
	continue
}
return dirName, true, nil
```

This preserves same-name reuse, allows same-stable matching-folder reuse, and lets different-name same-stable devices fall through to `makeUnique`.
